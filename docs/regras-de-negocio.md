# Regras de Negócio

1. Ativo crítico offline deve aparecer no dashboard.
2. Mudança de status de ativo gera histórico.
3. Item crítico de checklist não conforme pode abrir incidente automaticamente.
4. Chamado com prazo SLA expirado é classificado como fora do SLA.
5. Ações de escrita relevantes são auditadas.
6. Perfis de usuário controlam permissões por módulo.
